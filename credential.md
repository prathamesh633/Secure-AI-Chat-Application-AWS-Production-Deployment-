# Project Credentials and Placeholders Guide

This repository has been sanitized for security before pushing to Git. All specific AWS Resource IDs, Account IDs, OIDC domains, and Client Secrets have been replaced with generic placeholder values.

Below is a checklist of the files containing placeholder values and the corresponding values you should configure when deploying.

---

## Configuration & Script Placeholders

### 1. `deploy_cloudfront.py`
Contains CloudFront configuration settings.
* **`YOUR_VPC_ORIGIN_ID`**: The VPC Origin ID created when connecting CloudFront to your internal ALB (e.g., `vo_xxxxxx`).
* **`YOUR_ALB_INTERNAL_DNS`**: The internal DNS name of your EKS Application Load Balancer (e.g., `internal-k8s-xxxx.us-east-1.elb.amazonaws.com`).
* **`YOUR_AWS_ACCOUNT_ID`**: Your 12-digit AWS Account ID.
* **`YOUR_WAF_ACL_ID`**: The ID of the WAFv2 Web ACL created to protect the CloudFront distribution (e.g., `87f288c7-a9c9-...`).
* **`YOUR_ACM_CERTIFICATE_ID`**: The ID of the ACM SSL/TLS certificate for your custom domain.
* **`YOUR_DOMAIN_NAME`**: Your custom domain (e.g., `app.yourdomain.com`).

### 2. `redeploy_auth.py`
Lambda@Edge Auth0/OIDC deployment script.
* **`YOUR_CLOUDFRONT_DISTRIBUTION_ID`**: The ID of your CloudFront distribution (e.g., `E5ZISFWJ8P6GC`).
* **`YOUR_OIDC_ISSUER_DOMAIN`**: Your Auth0 or Okta tenant domain (e.g., `dev-xxxx.us.auth0.com`).
* **`YOUR_DOMAIN_NAME`**: Your custom domain (e.g., `app.yourdomain.com`).
* *Note: Pass the real `<CLIENT_ID>` and `<CLIENT_SECRET>` as CLI arguments when running this script.*

### 3. `deploy_lambda.sh`
Initial Lambda@Edge packager and deployer.
* **`YOUR_OIDC_ISSUER_DOMAIN`**: Your Auth0 tenant domain (e.g., `dev-xxxx.us.auth0.com`).
* **`YOUR_DOMAIN_NAME`**: Your custom domain (e.g., `app.yourdomain.com`).
* **`your-auth0-client-id`** / **`your-auth0-client-secret`**: Auth0 OIDC client credentials.

### 4. `setup_bastion.sh`
Bastion host bootstrap and setup script.
* **`YOUR_VPC_ID`**: The ID of your EKS cluster's VPC (e.g., `vpc-xxxxx`).
* **`YOUR_PUBLIC_SUBNET_ID`**: The public subnet ID where the bastion host will be placed.
* **`YOUR_CLUSTER_SECURITY_GROUP_ID`**: The security group ID of your EKS cluster control plane (to authorize bastion inbound traffic).
* **`YOUR_AWS_ACCOUNT_ID`**: Your 12-digit AWS Account ID.

### 5. `test_connection.py`
Connection validation utility.
* **`YOUR_BASTION_INSTANCE_ID`**: The EC2 Instance ID of the deployed SSM Bastion host.
* **`YOUR_API_GATEWAY_ID`**: The ID of your Private REST API Gateway.

### 6. `deploy_k8s.py`
Helper script to apply manifests to EKS via Bastion host.
* **`YOUR_BASTION_INSTANCE_ID`**: The EC2 Instance ID of the deployed SSM Bastion host.

### 7. `api_patch.json`
Resource policy payload for Private API Gateway.
* **`YOUR_AWS_ACCOUNT_ID`**: Your 12-digit AWS Account ID.
* **`YOUR_API_GATEWAY_ID`**: The ID of your Private REST API Gateway.
* **`YOUR_VPC_ENDPOINT_ID`**: The VPC interface endpoint ID (`execute-api`) allowed to invoke the API.

### 8. `r53_change.json`
Route 53 DNS record update payload.
* **`YOUR_DOMAIN_NAME`**: Your custom domain name (e.g., `app.yourdomain.com`).
* **`YOUR_CLOUDFRONT_DNS_NAME`**: The autogenerated domain name of your CloudFront distribution (e.g., `xxxx.cloudfront.net`).

## Application Secrets (Kubernetes / Local)

### 9. `Backedn/.env`
Local backend configuration environment file.
* **`AZURE_OPENAI_API_KEY`**: Your Azure OpenAI service API key.
* **`AZURE_OPENAI_ENDPOINT`**: Your Azure OpenAI endpoint URL (e.g., `https://your-resource.openai.azure.com/`).

### 10. `backend-secret.yaml`
Kubernetes Secret for the Backend pods.
* **`AZURE_OPENAI_API_KEY`**: Base64 encoded Azure OpenAI API key (or plain-text if using `stringData`).

### 11. `okta-oidc-secret.example.yaml`
Example Kubernetes Secret for Okta integration on the ALB Ingress.
* **`clientSecret`**: Your Okta client secret.
